> [데모 시연]

- **SAM (중간 원)**: $4.5B — 아시아 AI 고객지원 시장
- **SOM (가장 작은 원)**: $150M — 한국 중소기업 AI 고객지원
- **우측**: 연평균 성장률(CAGR) 그래프 — 25.8%

## 슬라이드 5: 제품

### 🎨 비주얼 설명
- **레이아웃**: 3개의 제품 기능 카드 (좌→우)
- **기능 1**: 🤖 "AI 자동 응답" — 채팅 위젯 스크린샷, "2초 응답, 85% 자동 해결"
- **기능 2**: 📊 "지능형 분석" — 대시보드 스크린샷, "감정 분석, 토픽 트렌드, CSAT 자동 측정"
- **기능 3**: ⚡ "AI 에이전트 (액션 수행)" — 워크플로우 다이어그램, "환불, 주문변경, 예약 — AI가 직접 처리"
- **기능 4**: 🎙️ "Voice-to-RAG" — 음성 녹음 업로드 → AI 지식베이스 자동 생성, "녹음만으로 AI 시작"
- **기능 5**: 📦 "데이터 허브 + 🕷️ 웹 크롤러" — HuggingFace 데이터셋 원클릭 + 홈페이지 자동 크롤링
- **하단**: 연동 로고 (카카오톡, 네이버, Slack, Shopify, 카페24)

### 🗣️ 토킹 포인트 (3분)

> "제품의 핵심 기능 다섯 가지를 소개합니다.
>
> **첫째, AI 자동 응답입니다.**
> 기업의 문서, FAQ, 상담 이력을 학습하여 고객 질문에 자동으로 답합니다.
> 단순 키워드 매칭이 아닙니다.
> '배송이 언제 와요?'라는 질문에 그 고객의 주문 상태를 조회해서
> '고객님의 주문 #12345는 내일 오후 도착 예정입니다'라고 답합니다.
> 개인화된 답변입니다.
>
> **둘째, 지능형 분석 대시보드입니다.**
> 모든 고객 대화를 실시간으로 분석합니다.
> 고객 감정 추이, 가장 많은 문의 유형, 불만족 패턴을 자동으로 발견합니다.
> 
> 예를 들어, '이번 주 환불 문의가 40% 증가했습니다. 
> 원인: 신제품 사이즈 가이드 부정확'과 같은 인사이트를 
> 고객지원팀이 아니라 제품팀에게도 제공합니다.
> 고객지원이 수동적 대응에서 능동적 개선으로 바뀝니다.
>
> **셋째, AI 에이전트입니다. 이것이 우리의 핵심 차별점입니다.**
> 기존 챗봇은 '답변'만 합니다. 우리의 Deep Agents는 '행동'합니다.
> LangChain 기반 멀티에이전트가 FAQ 검색, 주문 조회, 에스컬레이션을
> 각각 전문적으로 처리합니다.
> 고객이 '환불해주세요'라고 하면, AI가 주문을 조회하고,
> 환불 정책을 확인하고, 결제 시스템에 환불을 실행합니다.
> 상담원 개입 없이 30초 만에 완료됩니다.
>
> **넷째, Voice-to-RAG — 이것이 시장을 바꿀 기능입니다.**
> [데모 시연: 음성 파일 업로드]
> 소상공인은 FAQ 문서가 없습니다. 하지만 매일 전화 상담을 합니다.
> 그 통화 녹음을 업로드하면, AI가 자동으로 Q&A를 추출하고,
> 카테고리를 분류하고, 지식베이스에 등록합니다.

> **다섯째, 4대 데이터 수집 채널입니다.**

> 데이터 허브에서 도메인별 데이터셋을 원클릭으로 다운로드할 수 있습니다.

>
> 현재 카카오톡, 네이버 톡톡, 웹 채팅, Slack과 연동되어 있고,
> 카페24, Shopify와의 이커머스 통합도 완료했습니다."

## 슬라이드 6: 기술

### 🎨 비주얼 설명
- **중앙**: 기술 아키텍처 다이어그램 + 4대 데이터 수집 채널 강조
  ```
  ┌─────────────────────────────────────────────────────┐
  │           4대 데이터 수집 채널                        │
  │  🎙️ Voice-to-RAG │ 📦 데이터 허브                    │
  │  🕷️ 웹 크롤러    │ 📄 문서 업로드                    │
  └──────────────────┬──────────────────────────────────┘
                     ▼
  고객 질문 → 의도 분류 → RAG 파이프라인 → Deep Agents → 응답/액션
                              │                    │
                    ┌─────────┼─────────┐    ┌─────┴─────┐
                    │         │         │    │ LangChain │
                 벡터DB    지식그래프   실시간API │ 멀티에이전트│
                (기업 문서) (관계 이해) (주문/결제) └───────────┘
  ```
- **좌측**: "기존 챗봇" — 키워드 매칭, 정적 시나리오, 문서 필수 (X 표시)
- **우측**: "SupportAI" — RAG + Deep Agents + 4대 데이터 채널 (✓ 표시)
- **하단**: 기술 스택 아이콘 (Python, TypeScript, PostgreSQL, Pinecone, AWS, LangChain, Whisper)

### 🗣️ 토킹 포인트 (2-3분)

> "우리의 기술적 차별점에 대해 말씀드리겠습니다.
>
> 기존 챗봇은 두 가지 문제가 있습니다.
> 첫째, 시나리오 기반이라 새로운 질문에 대응 못 합니다.
> 둘째, 반드시 문서화된 FAQ가 있어야 합니다. 소상공인 대부분은 문서가 없습니다.
>
> SupportAI는 이 두 문제를 모두 해결합니다.
>
> **먼저 RAG (Retrieval-Augmented Generation)** 기술입니다.
> 기업의 모든 문서를 벡터화하여 실시간 검색하고, LLM이 정확한 답변을 생성합니다.
>
> **핵심은 4대 데이터 수집 채널입니다.**
> 
> 1. **Voice-to-RAG**: 통화 녹음을 업로드하면 Whisper STT로 텍스트 변환하고,
>    AI가 Q&A 쌍을 자동 추출합니다. 문서가 없는 소상공인도 바로 시작합니다.
> 2. **웹 크롤러**: 홈페이지 URL만 입력하면 FAQ, 정책, Q&A를 자동 추출합니다.
> 3. **데이터 허브**: HuggingFace에서 도메인별 데이터셋을 원클릭 다운로드합니다.
>    이미 507개 RAG 청크를 이커머스 도메인으로 구축 완료했습니다.
> 4. **문서 업로드**: 기존의 PDF, DOCX, CSV 파일도 당연히 지원합니다.
>
> 여기서 한 단계 더 나아갑니다. **Deep Agents**입니다.
> LangChain Deep Agents SDK 기반 멀티에이전트 아키텍처입니다.
> FAQ 에이전트, 주문조회 에이전트, 에스컬레이션 에이전트가 
> 각각 독립된 컨텍스트에서 전문적으로 작업하고 협업합니다.
>
